# DB layer (sqlc + pgx)

Typed SQL for the Vik Theatre API. Schema lives in `api/migrations/`; queries
are authored in `api/internal/db/queries/*.sql` and compiled by **sqlc** into
`api/internal/db/dbq/` (generated, do not hand-edit).

## Layout

```
api/
  sqlc.yaml                         # sqlc v2 config
  migrations/
    20260415000001_init.sql         # goose migration (schema)
    seed_dev.sql                    # idempotent dev seed (NOT a migration)
  internal/db/
    store.go                        # Store{Pool, *dbq.Queries}
    queries/
      users.sql batches.sql assets.sql
      consents.sql notes_rubric.sql audit.sql
    dbq/                            # generated by `make sqlc-gen`
```

## One-time install

```
brew install sqlc        # or: go install github.com/sqlc-dev/sqlc/cmd/sqlc@latest
brew install goose       # or the scoop/go-install equivalent
```

## Local dev workflow

```
cd api
make db-up                                         # postgres 16 in docker
export DATABASE_URL=postgres://vik:vik@localhost:5432/vik?sslmode=disable
make migrate-up                                    # applies goose migration
make sqlc-gen                                      # generates internal/db/dbq
make seed                                          # loads seed_dev.sql
make dev                                           # runs API
```

`seed_dev.sql` uses deterministic UUIDs and `ON CONFLICT DO NOTHING`, so it is
safe to re-run any time.

## Adding a new query

1. Pick the domain file (or create one) in `internal/db/queries/`.
2. Write SQL with a `-- name: <PascalCaseName> :<one|many|exec>` annotation.
   Use the `<Domain><Verb>` naming convention: `UserGetByPhone`, not
   `GetUserByPhone`.
3. Use `$1, $2, ...` placeholders (pgx style). Avoid `SELECT *` — list columns
   explicitly so sqlc generates stable struct fields.
4. Run `make sqlc-gen`. Commit both the `.sql` and the regenerated `dbq/`.

## Using the Store from a service

```go
store := db.NewStore(pool)
u, err := store.UserUpsertByPhone(ctx, "+91-9000000001")
```

For transactions, use `store.Pool.BeginTx` and build a per-tx `*dbq.Queries`
with `dbq.New(tx)`.

## Conventions

- UUIDs map to `github.com/google/uuid.UUID` via sqlc overrides.
- `timestamptz` maps to `time.Time`.
- `emit_interface: true` — inject `dbq.Querier` in tests for easy mocking.
- Never alter the schema file from a query; add a new goose migration instead.
